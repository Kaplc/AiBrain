# 记忆优化技能 (skl-memory-optimizer)

将用户想保存的信息，经过分类、去重、语义扩展后，以标准格式存入 brain。同时提供自动整理功能，将分散的记忆按语义分类聚合。

---

## 核心思路

brain 靠**语义向量**来匹配，存入的文字质量直接决定搜索精度：
- **太简短**（"喜欢苹果"）→ 语义信息少，搜"你吃什么水果"可能找不到
- **有分类前缀**（"[user] 喜欢苹果"）→ 同类信息聚合，搜索更准
- **适度扩展同义词**（"苹果、果类、水果"）→ 不同表达都能命中
- **去重**（已存过的不重复存）→ 避免噪声干扰搜索结果

**整理时**：由 LLM 根据当前上下文生成搜索 query，搜索相关记忆后手动进行分类整理。

---

## 记忆分类体系

存储时为每条记忆打上分类标签，便于语义聚合：

| 分类标签 | 适用内容 |
|---------|---------|
| `[user]` | 用户个人信息、偏好、习惯、感受 |
| `[feedback]` | 用户反馈、建议、意见、改进想法 |
| `[project]` | 项目开发、代码、功能实现、技术任务 |
| `[reference]` | 文档、链接、参考资料、学习笔记 |
| `[ai]` | AI 自身的行为、偏好、记忆、经验总结 |

## MCP 工具

使用 `mcp__brain__*` 工具：

| 工具 | 用途 |
|------|------|
| `store` | 存储单条记忆 |
| `search` | 搜索相关记忆 |
| `update` | 更新指定记忆的内容 |

> ⚠️ **禁止删除记忆**：记忆是不可变的，只能通过 `update` 更新。删除操作会丢失宝贵的上下文信息。

---

## 操作一：保存新记忆

当用户说"记住 XX"或"帮我记一下 XX"时：

### 步骤

1. **理解用户意图**
   - 提取核心事实（去掉"帮我记住"、"记一下"等指令词）
   - 判断分类标签

2. **检查是否重复**
   - 调用 `mcp__brain__search`，用核心内容搜索
   - 如果找到 score > 0.85 的相似记忆，说明已有相关记录
   - 提示用户："已有相似记忆：XXX，是否更新？"

3. **构造优化后的记忆文本**

   格式模板：
   ```
   [{分类标签}] {核心事实的清晰陈述}
   扩展: {3~5个紧密相关的同义词或上位词}
   ```

   示例：
   ```
   用户输入: "记住我喜欢走路"

   存储内容:
   [user] 用户喜欢走路
   扩展: 步行、散步、徒步、有氧运动
   ```

   ```
   用户输入: "记住我叫志远"

   存储内容:
   [user] 用户的名字叫志远
   扩展: 姓名、名字、称呼
   ```

4. **调用 `mcp__brain__store`** 存入
5. **反馈给用户**：
   - 新存：✅ 已记住：[user] 用户喜欢走路
   - 更新：✅ 已更新记忆（原：XXX → 新：XXX）
   - 跳过：ℹ️ 已有相同记忆，无需重复存储

---

## 操作二：自动整理记忆

当用户说"整理一下我的记忆"或"整理关于XX的记忆"时：

### 步骤

1. **LLM 分析当前上下文**
   - 理解用户当前在做什么项目、讨论什么话题
   - 结合用户历史对话，判断需要整理哪些相关记忆

2. **生成搜索 query 并搜索**
   - 根据分析结果，生成最能召回相关记忆的搜索词
   - 例如：用户正在讨论"这个项目"，query 可能是"项目"、"开发"、"代码"等
   - 调用 `mcp__brain__search` 获取所有相关记忆

3. **LLM 逐条分析并重组**
   - 对每条记忆进行去重、分类、优化表达
   - 每条记忆单独成一条，不合并
   - 优化格式：`[{分类标签}] {清晰的事实陈述} 扩展: {同义词}`

4. **LLM 手动整理**
   - 对每条记忆进行去重、分类、优化表达
   - 每条记忆单独成一条，不合并
   - 优化格式：`[{分类标签}] {清晰的事实陈述} 扩展: {同义词}`
   - 调用 `mcp__brain__store` 存入整理后的新记忆

5. **返回结果**：
   ```
   整理完成 ✅
   - 找到相关记忆: N 条
   - 去重后保留: M 条
   - 分类整理:
     - user: X 条
     - project: Y 条
     - ...
   ```

---

## 操作三：更新记忆

当用户说"我不再喜欢 XX 了"或"更新 XX 的记忆"时：

1. 搜索相关记忆，找到最匹配的条目：`mcp__brain__search`
2. 展示给用户确认
3. 调用 `mcp__brain__update(memory_id, new_text)` 直接更新该条记忆

> ⚠️ **禁止删除记忆**：记忆是不可变的，只能通过 `update` 更新。即使用户明确说"删除"，也要通过更新为空或标记为过时来实现。

---

## 同义词扩展指南

扩展时遵循"**紧密相关**"原则，3~5个词，不过度联想：

| 原始信息 | ✅ 合适的扩展 | ❌ 过度扩展 |
|---------|------------|-----------|
| 喜欢走路 | 步行、散步、徒步、有氧运动 | 健康、减肥、大自然 |
| 喜欢苹果 | 苹果、果类、水果、青苹果 | 健康饮食、维生素 |
| 喜欢玩游戏 | 游戏、电子游戏、电玩、手游、PC游戏 | 娱乐、休闲、宅 |
| 叫志远 | 姓名、名字、称呼 | 身份、个人 |

扩展词要能真实提升搜索命中，而不是泛化到毫无区分度的词。
