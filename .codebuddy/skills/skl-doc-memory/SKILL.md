# 文档记忆技能 (skl-doc-memory)

将纯文本文档智能分块存入 brain，并在需要时通过语义搜索召回内容或完整还原文档。

---

## 核心设计原则

brain 每条记忆本质上是一段文本，通过**向量语义相似度**来检索。所以：
- 文档要拆成小块，每块聚焦一个主题，语义搜索才准确
- 每块必须携带足够的元数据，这样才能把散块重新拼成完整文档
- 块大小建议 200~500 字，太大语义模糊，太小上下文丢失

---

## 操作一：存储文档

### 步骤

1. **解析文档结构**
   - 如果文档有标题（`#`、`##` 或明显的段落分隔），按标题/章节拆分
   - 如果是纯流水文本，按段落拆分（空行分隔），每块不超过 500 字
   - 如果段落太短（<50字），合并相邻段落

2. **为每块构造记忆文本**，格式如下：
   ```
   [DOC:{文档名}][{当前块序号}/{总块数}]
   {这一块的实际内容}
   ```
   例如：
   ```
   [DOC:项目设计方案][1/4]
   ## 背景
   本项目旨在解决用户数据分散的问题...
   ```

3. **依次调用 `mcp__brain__store`** 存入每一块
   - 存储时告知用户进度："正在存储第 X/N 块..."
   - 全部存完后汇总：共存入 N 块，文档名为 XXX

4. **存储索引块**（可选但推荐）
   对于超过 3 块的文档，额外存入一条索引记忆：
   ```
   [DOC_INDEX:{文档名}] 共{N}块，主要内容：{一句话摘要}
   ```

---

## 操作二：语义搜索

当用户想搜索某个主题或关键词时：

1. 调用 `mcp__brain__search`，用用户的查询词搜索
2. 过滤出 score > 0.7 的结果（相关性较高）
3. 展示结果时注明来源文档和块号：
   ```
   来源：项目设计方案（第2/4块）
   内容：...
   ```
4. 如果用户想看完整文档，告知可以用"完整召回"命令

---

## 操作三：完整召回文档

当用户要完整取回某篇文档时：

1. 先搜索 `[DOC_INDEX:{文档名}]` 获取总块数 N
   - 如果没有索引块，搜索 `[DOC:{文档名}]` 找到任意一块，从中读取总块数
2. 循环搜索每一块：
   - 搜索 `[DOC:{文档名}][{i}/{N}]`，i 从 1 到 N
   - 每次取 score 最高的结果
3. 按块序号排列，去掉元数据前缀，拼接成完整文档输出

> 提示：如果文档名不确定，先搜索关键词找到相关块，从块的元数据中提取文档名，再执行完整召回。

---

## 操作四：列出已存文档

当用户想知道存了哪些文档时：

1. 调用 `mcp__brain__search`，搜索 `DOC_INDEX`
2. 同时搜索 `DOC:` 以捕获没有索引的文档
3. 从结果中提取所有不重复的文档名，列表展示：
   ```
   已存储的文档：
   1. 项目设计方案（4块）
   2. 会议记录 2026-04-15（2块）
   3. ...
   ```

## 注意事项

- 文档名要简短有意义，方便后续召回，建议让用户确认
- 存储前询问用户文档名（如果用户没有提供）
- 如果文档很长（>10块），提前告知用户存储需要一点时间
- 召回时如果某块搜索不到，诚实告知用户该块可能丢失，并输出已找到的部分
