# 后端-stream 模块

## 概述
`stream.py` 是 AiBrain 后端的记忆流（活动流）模块，负责提供系统操作的历史记录查询功能。该模块从统计数据库中查询用户的记忆操作记录，支持按时间范围和操作类型过滤，为前端提供实时活动监控数据。

## 主要功能

### 1. 记忆流查询 API
提供 `/stream` 端点，支持灵活的历史记录查询：

#### GET `/stream`
- **功能**：获取记忆操作历史记录
- **查询参数**：
  - `action`：操作类型过滤（可选）
    - 支持的 action 值：`store`（存储）、`update`（更新）、`delete`（删除）等
  - `days`：按天数查询（优先使用）
    - 整数，最大30天
  - `limit`：按条数查询（days 未指定时使用）
    - 整数，最大200条，默认30条
- **返回**：JSON 格式的记忆流数据

## 查询模式

### 1. 按天数查询（优先）
- **参数**：`days=N`（N 为整数）
- **功能**：查询最近 N 天的操作记录
- **限制**：最大30天
- **示例**：`/stream?days=7` 查询最近7天的记录

### 2. 按条数查询（兜底）
- **触发条件**：`days` 参数未提供
- **参数**：`limit=N`（N 为整数）
- **功能**：查询最新的 N 条记录
- **限制**：最大200条，默认30条
- **示例**：`/stream?limit=50` 查询最新的50条记录

### 3. 操作类型过滤
- **参数**：`action=action_type`
- **功能**：只返回指定类型的操作记录
- **组合使用**：可与其他参数组合，如 `/stream?action=store&days=3`

## 返回数据结构

### 响应格式
```json
{
  "items": [
    {
      "id": "记录ID",
      "user_id": "用户ID",
      "action": "操作类型",
      "text": "操作内容",
      "created_at": "创建时间",
      "metadata": {}  // 附加元数据
    },
    // ... 更多记录
  ],
  "total": 123  // 符合条件的总记录数
}
```

### 字段说明
- **id**：记录的唯一标识符
- **user_id**：执行操作的用户ID
- **action**：操作类型
  - `store`：存储新记忆
  - `update`：更新现有记忆
  - `delete`：删除记忆
  - 其他自定义操作类型
- **text**：操作相关的文本内容（如记忆内容）
- **created_at**：操作时间戳（ISO 格式）
- **metadata**：附加信息（JSON 对象）
  - 可能包含：记忆ID、修改详情、操作结果等

## 数据库交互

### 1. 查询方法
模块调用 `stats_db` 对象的以下方法：

#### `query_stream_days(action, days)`
- **功能**：按天数查询操作记录
- **参数**：
  - `action`：操作类型（可选）
  - `days`：查询天数
- **返回**：指定天数内的操作记录列表

#### `query_stream(action, limit)`
- **功能**：按条数查询操作记录
- **参数**：
  - `action`：操作类型（可选）
  - `limit`：返回条数限制
- **返回**：最新的操作记录列表

#### `stream_count(action)`
- **功能**：统计符合条件的记录总数
- **参数**：
  - `action`：操作类型（可选）
- **返回**：总记录数

### 2. 数据表结构
操作记录存储在 `stream` 表中，典型结构：

```sql
CREATE TABLE stream (
    id TEXT PRIMARY KEY,
    user_id TEXT NOT NULL,
    action TEXT NOT NULL,
    text TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    metadata TEXT  -- JSON 字符串
);
CREATE INDEX idx_stream_created_at ON stream(created_at);
CREATE INDEX idx_stream_action ON stream(action);
CREATE INDEX idx_stream_user_id ON stream(user_id);
```

## 参数验证与错误处理

### 1. 参数验证
- **days 参数**：
  - 必须是整数
  - 自动限制最大30天：`min(int(days), 30)`
  - 无效值返回错误响应
- **limit 参数**：
  - 自动限制最大200条：`min(int(limit), 200)`
  - 默认值：30条
- **action 参数**：
  - 可选参数
  - 不做严格验证，由数据库处理

### 2. 错误响应
当参数无效时返回：
```json
{
  "error": "days 参数必须是整数",
  "items": [],
  "total": 0
}
```

### 3. 异常处理
- 数据库查询异常被捕获并返回空结果
- 确保 API 的稳定性

## 性能优化

## 使用示例

### 查询最近3天的所有操作
```bash
curl "http://localhost:5000/stream?days=3"
```

### 查询最近的存储操作
```bash
curl "http://localhost:5000/stream?action=store&limit=20"
```

### 查询特定用户的更新记录
```bash
# 需要前端或中间件添加用户过滤
curl "http://localhost:5000/stream?action=update&days=1"
```

### 获取统计信息（不返回详细记录）
```bash
# 只获取总数
curl "http://localhost:5000/stream?limit=0"
```

## 安全考虑

### 1. 权限控制
- 应验证用户访问权限
- 限制敏感操作记录的访问

### 2. 数据脱敏
- 敏感信息应在存储时脱敏
- 返回数据中隐藏敏感字段

### 3. 速率限制
- 防止恶意频繁查询
- 实施合理的 API 调用限制

## 注意事项

1. **数据量控制**：长时间范围的查询可能返回大量数据，需注意性能
2. **时区处理**：确保时间戳使用统一的时区
3. **数据一致性**：stream 表应与实际操作保持同步
4. **存储策略**：考虑历史数据的归档和清理机制
5. **监控告警**：监控 stream 表的增长和查询性能
