# 前端 - Logs 模块

## 概述
Logs 模块是 AiBrain 系统的日志查看器，用于实时监控系统日志、支持自动刷新、日志级别过滤和复制功能。该模块提供了一个用户友好的界面来查看和分析应用程序的日志文件。

## 主要功能

### 1. 日志查看
- **实时日志显示**：显示 app.log 文件中的日志内容
- **自动滚动**：新日志自动滚动到底部，确保看到最新日志
- **行号显示**：每行日志可点击复制

### 2. 日志过滤
- **级别过滤**：支持按 INFO、WARNING、ERROR 级别过滤日志
- **文本高亮**：不同日志级别使用不同颜色高亮显示
- **智能识别**：自动识别日志格式中的时间戳和级别标记

### 3. 自动刷新
- **可配置刷新间隔**：支持 1、3、5、10、30、60 秒的刷新间隔
- **自动刷新开关**：可启用/禁用自动刷新功能
- **手动刷新**：提供刷新按钮随时手动刷新日志

### 4. 复制功能
- **点击复制**：点击任意日志行可复制其内容
- **视觉反馈**：复制成功后显示短暂的提示信息

### 5. 设置管理
- **UI 设置保存**：自动刷新设置和间隔设置会自动保存到服务器
- **本地状态记忆**：页面刷新后保持之前的设置状态

## 技术实现

### HTML 结构 (`web/modules/logs/logs.html`)
- **主容器**：`.logs-wrap` 包含整个模块
- **标题栏**：`.log-header-row` 包含模块标题和设置按钮
- **工具栏**：`.log-toolbar` 包含刷新按钮和过滤选择器
- **日志查看器**：`.log-viewer` 包含日志标题和内容区域
- **设置弹出层**：`.settings-popover` 包含自动刷新设置

### JavaScript 功能 (`web/modules/logs/logs.js`)
- **`loadLogs()`**：从 `/logs` API 端点加载日志
- **`renderLogs(lines)`**：渲染日志行，应用颜色高亮
- **`filterLogs()`**：根据选择的级别过滤日志
- **`toggleAutoRefresh()`**：切换自动刷新开关
- **`saveLogSettings()`**：保存 UI 设置到服务器
- **`copyLogLine(row)`**：复制单行日志内容

### CSS 样式
- **暗色主题**：使用深蓝色系，符合整体 UI 风格
- **响应式布局**：适应不同屏幕尺寸
- **动画效果**：设置面板弹出动画、复制提示动画

## API 接口

### GET `/logs`
**参数**：
- `lines` (可选)：要获取的日志行数，默认 500 行

**响应**：
```json
{
  "file": "app.log",
  "total": 1500,
  "returned": 500,
  "lines": [
    "[2025-04-29 10:30:25] [INFO] 系统启动成功",
    "[2025-04-29 10:30:26] [WARNING] 内存使用率较高"
  ]
}
```

### GET `/ui-settings`
获取 UI 设置

**响应**：
```json
{
  "log_auto_refresh": true,
  "log_interval": 3
}
```

### POST `/ui-settings`
保存 UI 设置

**请求体**：
```json
{
  "log_auto_refresh": true,
  "log_interval": 5
}
```

## 使用场景

### 1. 系统监控
- 开发人员可以实时查看系统运行状态
- 运维人员可以监控错误和警告信息
- 测试人员可以查看测试过程中的日志输出

### 2. 故障排查
- 通过错误日志快速定位问题
- 通过时间戳分析事件发生顺序
- 通过日志级别过滤重点关注错误和警告

### 3. 开发调试
- 查看 API 调用记录
- 监控内存使用情况
- 跟踪用户操作流程

## 配置选项

### 自动刷新设置
- **启用/禁用**：控制是否自动刷新日志
- **刷新间隔**：1 秒到 60 秒的可选间隔
- **默认值**：自动刷新启用，间隔 3 秒

### 显示设置
- **默认显示行数**：500 行
- **颜色方案**：
  - INFO：绿色 (`#86efac`)
  - WARNING：黄色 (`#fde047`)
  - ERROR：红色 (`#fca5a5`)
  - 时间戳：灰色 (`#64748b`)

1. **搜索功能**：在日志中搜索特定关键词
2. **时间范围过滤**：按时间范围筛选日志

## 注意事项

1. **性能考虑**：默认限制显示 500 行日志，避免浏览器性能问题
2. **网络负载**：自动刷新间隔不宜过短，避免过多 API 调用
3. **安全性**：日志可能包含敏感信息，确保适当的访问控制
4. **文件大小**：大日志文件可能导致加载缓慢，建议定期归档
